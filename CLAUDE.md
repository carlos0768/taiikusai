# taiikusai - 体育祭パネル演出ソフト

## プロジェクト概要

体育祭のパネル競技を制御するためのWebアプリケーション。**タブレット（iPad）がメインデバイス。**

**パネル競技とは：**
スタンドに座っている観客全員がそれぞれ専用のパネル（冊子状、本のように開く形式で全色が1冊に入っている）を持ち、音楽に合わせてパネルを上げ下げすることで、遠くから見たときにスタンド全体がアニメーションに見える競技。体育祭の得点に影響する。

**パネルの色：黄色、赤色、黒色、青色、白色（折り目）の5色。**

**現状の課題：**
- Excelでセル一つ一つに色を打ち込み、別シーンも同じシートに書く運用
- シミュレーションはシートをスクロールして流れを掴む原始的な方法
- デザイン担当とExcel担当で作業が分離してしまう

**このソフトの目的：**
デザイン担当者がExcelの知識なしに、デザインからアニメーション確認まで簡単にできるツール。

## 最重要方針

このソフトは体育祭本番で使われる。パネルを上げるタイミングと位置のズレは、見栄えに直接影響する。得点にも関わるため、**正確性と信頼性が最優先**である。

- **タイミングの精度**: 音楽との同期がずれてはならない
- **位置の正確性**: 各パネルの上げ下げ指示が正しい位置に届かなければならない
- **本番での安定性**: 体育祭当日に支障なく動作すること。クラッシュ・遅延は許されない
- **シンプルさ**: 余計な機能より、核となる機能が確実に動くことを重視する
- **Excelより簡単**: デザイン担当者が迷わず使えること

## 技術スタック

- **Next.js (App Router)** + **TypeScript**
- **Supabase** — DB・認証・ストレージ
- **Tailwind CSS** — スタイリング
- **React Flow** — ダッシュボードの全体画面コンポーネント配置・連結
- **Canvas API** — グリッドエディタ描画（大量マス目のパフォーマンス確保）
- **@use-gesture/react** — タッチ操作（ピンチ、ドラッグ、長押し）
- **Claude Vision API (LLM)** — スキャン機能（手書きデザイン→マス目変換）
- **Vercel** — デプロイ

## 認証・プロジェクト

- **認証**: シンプルなユーザー名+パスワード認証（Supabase Auth）
- **プロジェクト機能**: 複数プロジェクト対応（複数チーム・複数年度で利用可能）
- **エクスポート**: 個人別パネル指示書の出力機能（仕様は後日決定）

## アプリ構成

### 1. ダッシュボード
- Supabaseホーム画面風UI（ダーク背景、ドット散りばめ、ピンチアウト可能）
- 全体画面コンポーネント（ウィジェット）がダッシュボード上に配置、ドラッグ移動可能
- 各コンポーネントにはグリッドの縮小サムネイル + 名前を表示
- 左上にハンバーガーメニュー（3本線）でサイドバー開閉
- 未連結の全体画面コンポーネントは右上に黄色い矢印アイコン表示
- 連結操作: 矢印を押してドラッグ → 直線が伸びていく → 別コンポーネントに繋ぐ
- 連結解除: 矢印の先端（繋がっている側）を繋がっている方向と反対側にドラッグすると外れる
- 分岐あり（1つから複数に繋げられる）
- コンポーネント削除: コンポーネントを長押し → 削除メニュー表示

### 2. 全体画面（パネルデザインエディタ）
- グリッドサイズはプロジェクト設定で指定可能（基本は横長）
- r/place風のワンタップ塗り：色を選択してセルをタップするだけ
- 使用可能色: 黄、赤、黒、青、白（プロジェクト設定で定義）
- **塗りつぶし（バケツ）**: 同色隣接セルを一括変更
- **範囲選択**: 矩形選択して一括塗り
- **Undo/Redo**: 操作の取り消し・やり直し
- 再生ボタンで連結先までアニメーション再生

### 3. ダッシュボード長押しメニュー（コンテキストメニュー）
- **スキャン**: カメラ起動 → LLMで手書きデザインをマス目に自動変換
- **手動**: 空の全体画面エディタを開く
- **テンプレ** (▶): 右に展開 → ユーザーが保存したテンプレート一覧から選択（プリセットなし）
- **既存** (▶): 右に展開 → 既存デザインを転用
- **インポート** (▶): 右に展開 → Excel/スプレッドシートを変換して取り込み

### 4. サイドバー
- 最上部: プロジェクト名
- ダッシュボード: メイン画面
- テンプレ: テンプレート管理（手動エディタと同じUI + 上部に検索ウィンドウ）
- 設定: 未定

### 5. アニメーション再生
- 連結された全体画面を順番に表示
- 遷移間隔はユーザーが調整可能（デフォルト値あり）
- パネル色変更プロセス（折る→次の色を出す）の時間を考慮
- 全体画面ページ内の再生ボタンから起動
- 分岐がある場合: 再生前に全ルートを一覧表示し、ユーザーが選んで再生
- 将来的に音楽ファイル取り込み・BPM同期の可能性あり

### 6. スキャン機能
- LLM（Claude Vision）で手書きデザイン画像をグリッドデータに変換
- 使用可能色（黄赤黒青白）へのマッピング
- フロー: スキャン → 結果を全体画面エディタに表示 → ユーザーが確認・微修正 → 確定
